# 受保护代码规则

## 总则

1. 未经明确授权，不允许修改受保护函数。
2. 受保护函数清单位于 `docs/STABLE_BASELINE.md` 的「当前禁止动的核心函数」章节。
3. 修改受保护函数必须经过三段审批：修改前报告 → 修改执行 → 修改后验证。

---

## 修改前必须输出

```
1. 计划修改文件：
2. 计划修改函数：
3. 是否触碰受保护代码：是/否
4. 为什么必须改：
5. 最小改动方案（具体到行号）：
6. 回归验证计划：
```

## 修改后必须输出

```
1. 实际修改文件：
2. 实际修改函数：
3. diff 摘要（改了几行、改了啥）：
4. 是否影响主流程：
5. 验证结果：
```

---

## 禁止行为

1. 禁止顺手优化。不允许在修 bug 的同时格式化代码、重命名变量、提取公共函数。
2. 禁止重构。不允许改变已有函数的结构、参数、返回值类型。
3. 禁止真实发布作为验证手段。验证必须使用 `--preview` 模式。
4. 禁止用临时脚本通过代替主流程通过。临时脚本通过不代表 `publish_pipeline.py` 能通过。
5. 禁止改了业务代码但不跑回归。修改后必须按 `docs/REGRESSION_CHECKLIST.md` 执行回归。

## 违反后果

1. 违反第 1-5 条任意一条，该次修改视为无效，需回滚。
2. 未执行回归验证就合并的代码，后续问题由修改者承担。
3. 触碰受保护函数未报告的，该次修改记录记入 bug 日志。
