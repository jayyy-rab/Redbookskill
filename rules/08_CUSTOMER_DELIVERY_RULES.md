# 08 — 客户交付规则

## 客户隔离

- 每个客户独立 `client_id`
- 不同客户的任务目录完全隔离
- 客户 A 不能看到客户 B 的截图、日志、结果

## 权限控制

客户可见：
- 任务状态
- 成功/失败账号
- 失败原因
- 截图
- 最终结果

客户不可见：
- Python 源码
- Chrome profile
- 账号密码
- cookies
- 核心规则文件
- 内部日志敏感字段
- 自动化细节

## 账号登录态

- 每个账号有状态记录：login_status、last_check_at、risk_status
- 未登录时明确返回"账号未登录，请人工维护登录态"
- 不要让客户以为是系统 bug

## 风控处理

遇到验证码、风控、扫码、异常登录：
- 不破解
- 不绕过
- 不继续自动点击
- 当前账号标记 `manual_review` 或 `failed`

## 内容合规

发布前检查：
- 极限词
- 虚假功效
- 医疗功效
- 低俗词
- 联系方式
- 诱导交易
- 敏感词

## 商品挂载合规

- 必须有商品名称匹配证据
- 必须有商品卡片截图
- 必须有商品 ID/名称证据
- 不匹配禁止 live 发布

## 重复内容风险

多账号发布时的防重复措施（预留，第一版可选）：
- 标题轻微变化
- 正文轻微变化
- 图片色调轻微变化
- 发布时间错开

## 任务取消

客户应能取消任务：
- Worker 每个步骤开始前检查 `ctx.state.cancel_requested`
- 检测到取消信号后优雅停止

## 断点续跑

- Step1-Step5 成功，可从 Step6 继续
- Step8 以后不建议直接续跑，必须重新打开页面复检

  "retry_count": 3,
