# 05 — 重试策略

## 核心规则

1. **只有 P2 错误触发重试** — P0/P1 不重试
2. **重试耗尽后降级为 P1** — 不再重试，停止当前账号
3. **重试次数累加** — 每次重试 `retry_count += 1`
4. **幂等重试** — 重试必须安全可重复，不能产生副作用（重复扣费、重复发布）

## 完整重试表

见 `docs/ENGINEERING_FLOW_V1_1.md` 第 八 节「retry_policy 总表」。

该表覆盖：Chrome/CDP 连接、页面加载、登录等待、画图软件登录等待、生图等待、图片上传、标题填写、正文填写、商品搜索、商品卡片挂载、发布等待、文案模型调用、文案重写、参考图下载、prompt 回读。
