# 07 — AI 修改规则

## 修改前必须读

1. `rules/ENGINEERING_BASELINE.md`（v1.1 基线）
2. `rules/00-INDEX.md`（规则索引）
3. `docs/ENGINEERING_FLOW_V1_1.md`（工程逻辑图，冻结版）
4. 当前失败日志
5. 当前失败截图
6. `bugs.json`

## 禁止事项

1. 不允许改变量名
2. 不允许改 `input.json` 结构
3. 不允许改 `result.json` 结构
4. 不允许改目录结构
5. 不允许删除校验
6. 不允许为了通过测试而跳过失败
7. 不允许默认点击发布
8. 不允许修改非当前步骤代码
9. 不允许顺手重构其他模块

## 修改要求

1. 改完必须单步验证
2. 单步验证通过后再跑全链路 preview
3. 必须写证据（截图/日志）
4. 必须更新 `StepResult`

## 输出格式

每次修改前输出：
```
【根因分析】
【涉及文件】
【计划修改点】
【不会修改的范围】
【验证命令】
```
